Biodegradability Classification

Welcome to Biodegradability Classification

The goal of our module is to teach the user about the biodegradability of organic chemicals through machine learning and data science principles.

Our module gives the user all of the options and tools needed to build a machine learning model, step-by-step. This includes selecting the data and type of algorithm to use, fine-tuning hyperparameters, evaluating the model's performance, and feeding the model new, unseen instances of organic chemicals, which the model will be able to predict and classify as "readily biodegradable" or "not readily biodegradable."

By giving the user these tools, they will be able to see what chemical properties are highly correlated to biodegradability, and which machine learning and data science practices help improve this model's accuracy.

About our dataset

Class:

This is the value we are training the model to find. It is a binary representation of biodegradability, where...

  • Ready biodegradability (> 60% biodegradation within 28 days) = (1)
  • Non-ready biodegradability (< 60% biodegradation within 28 days) = (0)

SMILES String:

  • SMILES String: A code representation of the organic chemical's formula/structure.

Fingerprints:

  • Fingerprint: A binary representation of the molecule's substructures.
  • For training their model, the user will have the choice between three fingerprints:
    • Morgan Fingerprint
    • ECFP2
    • Path Fingerprint

Features:

  • Features: A list of over 200 molecular/electronic properties, including...
  • For training their model, the user will have the choice between three fingerprints:
    • molecular weight
    • number of valence electrons
    • number of aromatic rings
    • H Donor vs. Acceptor
    • etc.

NOTE: The data table shows only a preview of 15 of the chemicals in the dataset. In reality, the dataset contains over 6000 chemicals. Preparing Data

Before training can begin, several decisions must be made in terms of preparing data, and choosing the type of algorithm:

Selecting/Dropping Data:

Choosing which properties will or won't be used to train the model is an important step in data preparation.

While one might assume that using more properties would lead to better accuracy, this is not necessarily the case. This is referred to as the curse of dimensionality, where including too much data can worsen the decision-making of Machine Learning algorithms.

The user should keep this in mind when choosing whether to use features or one of the fingerprints to train their model.

Splitting Data:

The next stage, splitting data can be broken down into three steps:

  • Training: Helps the model learn patterns and relationships within the data.
  • Validation: Tunes the model's parameters and selects the best mode, generalizing the data well.
  • Testing: Evaluates the final model's performance on entirely unseen data.

Classification vs. Regression:

Classification and Regression are both supervised learning algorithms (learning from datasets to predict outcomes), however, there is a difference:

  • Classification: Determined by predefined classes (color, size, etc.)
  • Regression: Determined by numerical data (prices, temperature, etc.)

Because our module is focused on the binary property of ready biodegradability, we will be using Classification.

Our Algorithms

For this assessment, you will have the choice between 3 Machine Learning algorithms, each of which has its own strengths and weaknesses:

  • Decision Tree (DT): Flowchart-like structure, narrows down choices given a large sample.
    • Pros: Simple, handles numerical & categorical data
    • Cons: Small changes can largely impact data
  • K-Nearest Neighbor (KNN): Compares input to rest of data, determining which data the input is most similar to.
    • Pros: Non-parametric, versatile
    • Cons: Memory intensive, exhibits curse of dimensionality
  • Support Vector Classification (SVC): Separates classes using a line.
    • Pros: Handles large datasets, memory efficient
    • Cons: Computationally expensive, sensitive to noisy data

Hyperparameter Tuning

Hyperparameter tuning is the process of finding the best set of hyperparameters for a machine learning algorithm by selecting the optimal values to improve the model's performance. However, be careful of overfitting. This occurs when the model essentially "memorizes" its training data, resulting in higher accuracy values for training data, but lower accuracy for new data. Depending on your chosen ML algorithm, you will be able to tune the following hyperparameters:

For Decision Tree (DT):

  • Max depth of tree
    • How far the tree is allowed to go
    • Higher depths can fit data better, but can also overfit data
  • Splitter strategy:
    • Best
      • Chooses best split at each node
    • Random
      • Chooses best random split at each node

For K-Nearest Neighbor (KNN):

  • Number of neighbors
    • How many neighbors near a point will be used to classify that point
    • Fewer neighbors can fit data better, but k=1 will overfit data
  • Weights
    • Uniform
      • All nearby points are weighted equally
  • Distance
    • Closer neighbors are weighted more than further ones

For Support Vector Classification (SVC):

  • Regularization Parameter
    • How smooth the decision boundary will be
    • Higher values can fit data better, but can also overfit data
  • In general, SVCs are the most resistant to overfitting
